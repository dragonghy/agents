## Lessons Captured

1. **pitfall #12 → `claude.md`**: After `gh pr merge`, always `git pull --rebase origin main`. Otherwise anything reading files from the working tree runs pre-merge code.
2. **Stashed fixes are evidence a prior session hit the same wall.** When I stashed "unrelated" changes at the start of the session, I didn't check them — the very bug that blocked the daemon restart was sitting in the stash. Check stashes before doing anything that rebuilds infra.
3. **`gh pr merge --delete-branch` errors when local branch is checked out in a worktree.** The remote merge still succeeds, but the error message is misleading ("failed to run git"). Next time, `git worktree remove` first, then merge.
