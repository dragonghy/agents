## Operational lessons captured

1. **Branch race when parallel subagents share a worktree.** Phase 4 Telegram subagent reported repeatedly being stashed and switched to `chore/v1-infra-cleanup` branch by the parallel cleanup subagent. Always isolate each parallel subagent into its own `git worktree add /private/tmp/agents-<task>` before spawning. Tracked as task #41.
2. **`pytest | tee | tail -X` hang.** Across ~9 rounds of subagent test runs, pytest python processes leaked in state=S after writing "X passed" — wrapper never returned because the python process never exited. Killed manually each time via `pkill -9 -f 'uv run pytest'`. Root cause likely asyncio fixture cleanup leaks (esp. `test_orchestration_events.py` SSE generator). Tracked as #39 (workflow workaround) and #40 (diagnosis).
3. **`ScheduleWakeup` is unreliable for long autonomous runs.** Switched to `Monitor` with `until-loop` to stay alive 55 min in a single turn. Pattern: `Monitor(command="START=$(date +%s); while true; do ...; sleep 60; done", timeout_ms=3500000)`. The runtime caps `timeout_ms` at 3,600,000 (60 min) so chunking is required for longer runs.
4. **Self-merge criterion for cleanup PRs**: only merge without Human review if (a) tests stay green throughout, (b) diff is purely deletions plus regression-free pruning, (c) commits match the brief spec line-by-line. PR #33 cleared all three; PRs #31/#32 introduce new code paths or substantive content and went to Human review.
