## Key Discussion: Retrospective & Redesign

Human identified the core problem with v1: **context switch cost is too high**. They can't manage 13 projects and 18 agents simultaneously. The single most important decision: **focus 100% on Agent Harness itself**.

### Architecture Decision: Agent-Centric → Task-Centric
Human wrote a comprehensive RETROSPECTIVE.md with the following key insights:

1. **Memory is broken**: Agent context lost on restart. Multiple instances (dev-alex, dev-emma) was a workaround that didn't work. Solution: externalize memory into tickets, docs, skills.

2. **Agent is a template, not an entity**: No persistent identity needed. Agent type = system prompt + tools + permissions.

3. **Task is the scheduling unit**: System dispatches tasks, not agents. Ephemeral sessions spawned on demand.

4. **3 Agent Types replace 18 instances**: Development (plan/implement/test/deliver), Operations (health/infra/config), Assistant (personal tasks).

5. **Morning Brief Protocol**: Daily async communication. Status snapshot + decisions needed + resources needed. Human responds with natural language.

## Decisions Made
1. Focus 100% on Agent Harness
2. Freeze all other projects
3. V2 architecture with ephemeral agents
4. Admin = COO, Human = Chairman
5. 3 agent types (development, operations, assistant)
6. Morning Brief as primary communication
