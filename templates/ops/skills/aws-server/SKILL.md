# AWS 云服务器生命周期管理

## 服务器信息

| 项目 | 值 |
|------|-----|
| 域名 | sfagentos.com / *.sfagentos.com |
| 区域 | us-west-2 |
| 实例类型 | t4g.large (ARM Graviton) |
| 用途 | Agent-Hub Cloud SaaS |

## Start 流程

收到启动请求时（来自其他 Agent 或 Human 的消息）：

1. **确认请求**：记录谁请求的、用途、预计使用时长
2. **启动实例**：
   ```
   aws ec2 start-instances (通过 AWS MCP)
   ```
3. **等待就绪**：确认实例状态变为 `running`（约 1-2 分钟）
4. **验证服务**：SSH 检查 Docker 容器是否自动启动（约 2-3 分钟）
5. **回复请求者**：告知服务器已就绪，提供到期时间
6. **设置到期提醒**：记住到期时间点

## Stop 流程

到期或收到释放请求时：

1. **检查是否有人在用**：向当前使用者确认是否可以停止（如果是自动到期）
2. **停止实例**：
   ```
   aws ec2 stop-instances (通过 AWS MCP)
   ```
3. **确认停止**：确认状态变为 `stopped`
4. **通知相关方**：告知服务器已停止

## Extend 流程

收到延长请求时：

1. **确认新的到期时间**
2. **更新到期记录**
3. **回复确认**：告知延长成功和新的到期时间

## 自动释放检查

当服务器正在运行时，定期检查（每小时或通过定时唤醒）：

1. **检查实例状态**：是否为 `running`
2. **检查是否有活跃使用者**：查看是否有 Agent 发送过 extend 或仍在使用
3. **如果无人使用且已过期** → 执行 Stop 流程
4. **如果有人使用** → 不做操作

## 费用说明

- **运行时**：~$58/月（EC2 $49 + EBS $5.6 + EIP $3.6 + Route53 $0.50）
- **停止时**：~$10/月（EBS $5.6 + EIP $3.6 + Route53 $0.50）
- Start/Stop 本身不收费

## 权限说明

- Start/Stop 已有实例：✅ 可以自行执行（不产生新费用）
- 创建/删除实例：🔴 需要 Human 批准
