## 1. 邀请码系统设计

### 1.1 核心概念

每位宾客/家庭生成一个**唯一邀请码**，对应一个专属链接。宾客通过此链接访问网站并完成 RSVP。

**链接格式**：`https://snowflower.love/invite/{CODE}`

例如：
- `https://snowflower.love/invite/ZHANG2026` → 张叔叔一家
- `https://snowflower.love/invite/LILY0523` → Lily & Tom

### 1.2 访问控制策略

**推荐方案：公开网站 + 邀请码仅用于 RSVP**

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **A. 公开网站 + 邀请码 RSVP（推荐）** | 任何人可浏览网站，但 RSVP 需要邀请码 | 方便分享、搜索引擎友好、宾客可转发给家人看 | 非受邀者能看到婚礼信息（通常不是问题） |
| B. 全站密码保护 | 必须输入邀请码才能进入网站 | 隐私性强 | 分享不便、每次访问都要输码、微信分享体验差 |
| C. 全部公开无邀请码 | 网站和 RSVP 都公开 | 最简单 | 可能收到无关人的 RSVP |

**方案 A 的理由**：
- 微信分享时，宾客点开就能看到精美的婚礼页面，体验更好
- 邀请码在 RSVP 环节才需要，自然筛选真正受邀的人
- 宾客可以把网站链接转发给家人一起看，不需要每人都有邀请码

### 1.3 邀请码规则

- **格式**：4-8 位大写字母 + 数字，易于手动输入
- **生成规则**：可以用宾客姓名拼音首字母 + 随机数，也可以纯随机
- **一码多人**：一个邀请码对应一个家庭/组，RSVP 时填写出席人数
- **大小写不敏感**：输入时自动转大写

### 1.4 邀请码链接行为

```
宾客点击 snowflower.love/invite/ZHANG2026
  ↓
网站首页打开，自动识别邀请码
  ↓
导航栏显示 "欢迎，张叔叔一家"
  ↓
进入 RSVP 页面时，姓名自动预填
  ↓
提交 RSVP，关联到该邀请码
```

## 2. 数据存储方案

### 2.1 数据库表设计（Supabase）

**guests 表（宾客名单）**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| invite_code | varchar(8) | 唯一邀请码 |
| name_zh | varchar | 中文名 |
| name_en | varchar | 英文名 |
| group_name | varchar | 组/家庭名（如"张叔叔一家"）|
| max_guests | int | 最大出席人数 |
| email | varchar | 邮箱（可选） |
| phone | varchar | 手机号（可选） |
| wechat | varchar | 微信号（可选） |
| channel | varchar | 发送渠道（email/wechat/sms） |
| invited_at | timestamp | 邀请发送时间 |
| created_at | timestamp | 创建时间 |

**rsvp 表（在现有基础上扩展）**：

| 新增字段 | 类型 | 说明 |
|---------|------|------|
| invite_code | varchar(8) | 关联邀请码（可选，兼容无码提交） |

### 2.2 管理后台扩展

现有 /admin 页面需要扩展：

- **宾客名单管理**：添加/编辑/删除宾客，生成邀请码
- **批量导入**：上传 CSV（姓名、邮箱、手机、分组）
- **RSVP 状态追踪**：已邀请 / 已查看 / 已回复 / 未回复
- **统计看板**：总邀请数、已回复数、确认出席数、总人数

## 3. 请帖发送方案

### 3.1 发送渠道

| 渠道 | 适用人群 | 方式 |
|------|---------|------|
| **微信** | 国内亲友（主要渠道） | 分享链接 + 精美预览卡片 |
| **邮件** | 海外朋友、同事 | HTML 邮件 + 邀请链接 |
| **短信** | 长辈（不常用微信） | 简短文字 + 链接 |
| **纸质请帖** | 重要长辈 | 印有 QR Code，扫码进入专属链接 |

### 3.2 微信请帖方案（最重要）

**微信分享体验优化**：
- 设置 Open Graph 和微信分享 meta 标签
- 分享卡片显示：婚纱照缩略图 + "华阳 & [伴侣] 婚礼邀请" + snowflower.love
- 宾客点击即打开网站首页

**微信请帖内容模板**：

```
🌸 华阳 & [伴侣名] 诚挚邀请您

参加我们的婚礼

📅 2026年5月23日
📍 哈佛姆贝

请点击链接查看详情并回复出席：
https://snowflower.love/invite/{CODE}

期待与您相聚！💕
```

### 3.3 邮件请帖模板

**主题**：华阳 & [伴侣] 婚礼邀请 | Wedding Invitation

**正文**（中英双语 HTML 邮件）：
- 顶部：婚纱照 banner
- 中间：婚礼信息（日期、地点、日程）
- CTA 按钮："查看婚礼详情 & RSVP" → 指向专属链接
- 底部：联系方式

### 3.4 QR Code 方案

- 每个邀请码生成一个 QR Code
- QR Code 指向 `snowflower.love/invite/{CODE}`
- 可打印在纸质请帖上
- 也可在微信中作为图片分享

## 4. 实现优先级

### Phase 1（M4 同步，优先）
1. guests 表 + 邀请码生成逻辑
2. `/invite/{CODE}` 路由 → 首页 + 自动识别宾客
3. RSVP 表单关联邀请码 + 姓名预填
4. 管理后台：宾客名单 CRUD + RSVP 状态

### Phase 2（上线前）
5. 微信分享 meta 标签优化
6. 邮件请帖 HTML 模板
7. QR Code 生成（每个邀请码）
8. 批量导入宾客名单（CSV）

### Phase 3（上线后）
9. 发送追踪（已发送/已查看/已回复）
10. 管理后台统计看板
11. 提醒功能（未回复宾客二次提醒）

## 5. 技术实现建议

- **邀请码路由**：Next.js dynamic route `/invite/[code]/page.tsx`
- **邀请码存储**：cookie/localStorage，首次通过 `/invite/{CODE}` 进入后记住，后续直接访问也能识别
- **QR Code 生成**：`qrcode` npm 包，管理后台中生成可下载 PNG
- **邮件发送**：可用 Outlook MCP 或 Resend/SendGrid API
- **微信分享**：设置 `og:title`, `og:description`, `og:image` meta 标签
